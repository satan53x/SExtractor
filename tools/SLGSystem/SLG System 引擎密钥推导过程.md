# SLG System 引擎密钥推导过程

## 封包格式（所有游戏相同）

`script.szs` 文件头为明文：

```
SZS100__  00 00 00 00  0B 00 00 00  ...
签名(8B)   version(4B)  count(4B)
```

索引区每条目 **272 字节**（256B 名称 + 8B offset + 8B size），索引区明文，数据区加密。

---

## 加密层结构

SZS 数据区有两层加密，解密顺序从外到内：

**第一层（所有游戏相同）：** 每字节 XOR 0x90

**第二层：** LCG 密钥流，但运算方式因游戏而异（见下方各游戏分析）

LCG 算法（两种模式共用）：

```python
x = seed & 0xFFFFFFFF
x = (x * 0x343FD + 0x269EC3) & 0xFFFFFFFF
ks[i] = (x >> 16) & 0xFF     # 取第 16~23 位
```

---

## 天極姫系列（XOR 模式）

### 算法

```python
# 解密（XOR 对称，加解密相同）
plain[i] = (stored[i] ^ 0x90) ^ ks[i]
```

### 密钥推导方法：已知明文攻击

**关键观察：** XOR 0x90 解包后，ev / lbn / sfn 三个文件在偏移 **16~32** 处字节完全相同。

这三种文件的条目结构为 `[64B 名称 + 4B offset]`，第一条目的名称通常很短（如 `m_batyou01`，10 字节），其余 54 字节为 `\x00` 填充，因此偏移 16~64 范围明文全零。

明文为零时：

```
cipher[i] = 0x00 ^ ks[i] = ks[i]
```

**密文本身就是密钥流。** 偏移 16~32 对应密钥流的第 17~32 字节。

**反推 seed：**

1. 枚举所有满足 `k17 & 0xFF == ks[0]` 的 15 位值，共 128 个候选
2. 逐一验证后续 15 步是否匹配，唯一确定 `k17`
3. 用模逆元反推 seed：

```
k_prev = ((k_next << 16 | low16) - B) * A_inv % M
其中 A=0x343FD, B=0x269EC3, M=0x100000000
A_inv = modinv(A, M)
```

枚举 low16（0~65535）得到 65536 个候选 seed，**解密结果完全相同，取任意一个即可。** 原因是 LCG 只取第 16~30 位，seed 的低 16 位对密钥流无影响。

### 已知密钥

| 游戏 | seed |
|---|---|
| 天極姫 ～新世大乱・双界の覇者達～ | `0x1291f641` |
| 天極姫2 | `0xe41ef641` |

两作低 16 位 `0xf641` 完全相同，推测同一编译环境的构建特征。对同系列未知游戏可优先固定低 16 位只暴力高 16 位（65536 次，毫秒级）。

## 三極姫 Renewal（减法模式）

### 算法

```python
# 解密
plain[i] = (stored[i] ^ 0x90) - ks[i]    # 减法

# 加密（逆过程）
stored[i] = (plain[i] + ks[i]) ^ 0x90    # 加法
```

### 密钥来源

seed `0x7F501E37` 由瑜瑜提供的工具 `szs_common.py` 给出，验证后确认正确。

### 为何已知明文攻击在此失败

三極姫 ev/lbn/sfn/swn 四个文件在偏移 8~64 处密文也完全相同（同样是零填充区域），理论上可以用相同思路推导，但实际推导失败，原因是算法假设错误：

整个推导过程假设第二层是 XOR 对称操作，因此将密文直接当作密钥流代入验证。而实际算法是减法，零明文下：

```
stored[i] = (0 + ks[i]) ^ 0x90 = ks[i] ^ 0x90
```

真正的密钥流 = `stored[i] ^ 0x90`，而非 `stored[i]`。做了 XOR 0x90 解包之后拿到的才是密钥流，这一步实际是对的——但后续用 XOR 方程验证候选 k 仍然是错的，因为减法模式下 `0 - ks ≠ 0`，所有候选全部不通过。

## 工具使用

```bash
# 天極姫系列（xor 模式，默认）
python szs_tool.py unpack script.szs ./out 0x1291f641
python szs_tool.py pack   ./out script_new.szs 0x1291f641

# 三極姫（sub 模式）
python szs_tool.py unpack script.szs ./out 0x7f501e37 --mode sub
python szs_tool.py pack   ./out script_new.szs 0x7f501e37
# pack 时 mode 从 _order.txt 自动读取，无需重复指定
```
